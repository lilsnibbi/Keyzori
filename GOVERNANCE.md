# Governance

Keyzori is maintained in the open. Changes are proposed through issues and pull requests, reviewed for architecture, tests, compatibility, documentation, and operational impact, and accepted by repository maintainers.

Maintainers may merge focused changes after review and required checks pass. Breaking changes require a public proposal and follow the [release policy](.github/RELEASE_POLICY.md). Security reports use the private process in [SECURITY.md](SECURITY.md). Contributor behavior is governed by [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).

Contributor access is earned through sustained, constructive participation. Maintainer changes and material project-direction decisions should be recorded publicly when security or privacy does not require confidentiality.
